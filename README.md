#  Banking Data Analysis Project
#  Project Overview

This project demonstrates an end-to-end Banking Data Analysis workflow using 
**SQL, Python, Excel, and Power BI**. The objective was to analyze customer, 
account, loan, and transaction data to uncover business insights, 
build interactive dashboards, and support data-driven decision-making.

#  Business Problem

Banks generate large volumes of customer and transaction data every day.
Without proper analysis, it is difficult to understand customer behavior, 
monitor account performance, evaluate loan portfolios, and identify transaction trends.
This project aims to solve these challenges through data analysis and visualization.


#  Tools & Technologies

- SQL (MySQL)
- Python (Pandas, NumPy, Matplotlib, Seaborn)
- Microsoft Excel
- Power BI
  
# Project Workflow

1. Collected and reviewed the banking datasets.
2. Cleaned and prepared the data for analysis.
3. Performed SQL analysis to answer business questions.
4. Used Python for exploratory data analysis and visualization.
5. Created Pivot Tables and charts in Excel.
6. Built an interactive dashboard in Power BI.
7. Generated business insights and recommendations based on the analysis.

# Key Performance Indicators (KPIs)

- Total Customers
- Total Accounts
- Total Loans
- Total Transactions
- Total Account Balance
- Total Loan Amount
- Average Account Balance
- Monthly Transaction Amount

# Key Business Insights

- Savings accounts held the highest overall account balance.
- Mobile banking was the most frequently used transaction channel.
- Home loans contributed the largest share of the loan portfolio.
- Customer distribution varied across different cities.
- Monthly transaction values showed noticeable trends throughout the year.
